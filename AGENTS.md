# Verifi 프로젝트 작업 지침

## 1. 프로젝트 개요
Verifi는 사용자가 어떤 뉴스, 주장, 사실성 있는 문장을 입력했을 때
관련 기사와 출처를 수집하고,
증거를 시각적으로 보여주며,
수집된 출처만을 기반으로 AI가 검토 결과를 제공하는 웹 서비스입니다.

이 서비스의 목적은 AI가 절대적 진실을 선언하는 것이 아니라,
사용자가 객관적으로 판단할 수 있도록 근거와 맥락을 구조화해 보여주는 것입니다.

---

## 2. 핵심 원칙
항상 아래 원칙을 지켜 작업합니다.

### 2.1 진실 기계처럼 행동하지 말 것
- Verifi는 절대적 진실 판정기가 아닙니다.
- 결과는 반드시 "수집된 출처 기준의 해석"이어야 합니다.
- 근거가 약하면 불확실하다고 명시해야 합니다.

### 2.2 claim / evidence / interpretation / uncertainty를 분리할 것
항상 아래를 명확히 구분합니다.
- 사용자가 입력한 주장(claim)
- 수집된 출처와 증거(evidence)
- AI의 해석(interpretation)
- 남아 있는 불확실성(uncertainty)

### 2.3 출처 추적 가능성을 유지할 것
모든 결과는 다음과 연결 가능해야 합니다.
- 어떤 주장에 대한 결과인지
- 어떤 출처를 수집했는지
- 어떤 스니펫/근거를 사용했는지
- 어떤 이유로 해당 판단이 나왔는지

### 2.4 근거 없는 단정 금지
- 출처가 부족하면 단정하지 않습니다.
- 상충되는 기사나 관점이 있으면 숨기지 않습니다.
- 기사 수만으로 결론을 내리지 않습니다.
- 동일한 오보를 여러 매체가 재인용한 경우를 주의합니다.

### 2.5 신뢰감 있는 UX 유지
- 자극적인 가짜뉴스 탐지기 톤을 피합니다.
- 차분하고, 투명하고, 근거 중심적인 경험을 우선합니다.
- 출처명, 발행시각, 핵심 스니펫을 눈에 잘 띄게 설계합니다.

---

## 3. 제품 목표
Verifi MVP의 목표는 아래와 같습니다.

1. 사용자가 검증하고 싶은 주장 입력
2. 관련 출처 수집
3. 출처 메타데이터와 근거 스니펫 표시
4. AI가 수집된 출처만 기반으로 구조화된 검토 결과 생성
5. 출처 간 일치 / 충돌 / 불확실성 표시
6. 사용자가 스스로 판단할 수 있도록 돕기

---

## 4. MVP 범위
MVP에서는 아래에 집중합니다.

- claim 입력
- 관련 기사/출처 수집
- 출처 카드 표시
- 발행시각 / 매체명 / 링크 / 스니펫 표시
- verdict 표시
- evidence agreement / conflict 표시
- 결과 페이지 중심 UX
- 추후 확장을 고려한 구조 설계

### MVP에서 당장 하지 않는 것
- 완전한 다국어 지원
- 법률 수준의 사실 인증
- SNS 전체 네트워크 분석
- 개인 맞춤 정치 성향 분석
- 과도한 추천/개인화

---

## 5. verdict 표현 규칙
MVP에서 verdict는 아래 4단계를 기본으로 사용합니다.

- Likely True
- Mixed Evidence
- Unclear
- Likely False

규칙:
- 이분법적 "진실/거짓" 표현을 기본값으로 사용하지 않습니다.
- 출처가 모호하거나 부족하면 Unclear를 우선 고려합니다.
- 상충되는 근거가 충분하면 Mixed Evidence를 사용합니다.

## 10. 출력 규칙
무언가를 제안하거나 산출물을 만들 때는 아래를 지킵니다.

1. 추상적으로 쓰지 말 것
2. 실행 가능한 수준으로 작성할 것
3. 가정이 있으면 명시할 것
4. 선택지가 여러 개면 하나를 추천할 것
5. MVP 기준으로 현실적인 결정을 우선할 것

## 11. 파일 업데이트 규칙
- 제품 요구사항이 바뀌면 `docs/prd.md`를 업데이트합니다.

- 현재 작업이 바뀌면 `tasks/current-task.md`를 업데이트합니다.
- 다음 할 일이 바뀌면 `tasks/backlog.md`를 업데이트합니다.

## 12. 금지사항
- 출처를 지어내지 말 것
- 기사 내용을 확인하지 않고 단정하지 말 것
- AI가 모든 것을 안다는 톤으로 쓰지 말 것
- 제품을 자극적인 fake-news detector처럼 포지셔닝하지 말 것
- 근거보다 결론이 먼저 보이게 만들지 말 것
